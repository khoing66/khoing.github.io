基于对话框编程

模态对话框

  1. 创建基于对话框的MFC应用程序框架

    1. 选择基于对话框 – 完成
    2. 视图—资源视图–dialog–第二个

  2. ui中右击属性进行设置

  3. 工具箱插入按钮，改名字不要双击，直接选中，进行编辑就行

  4. 右击dialog插入dialog，创建另外一个对话框。右击属性，可以更改id

  5. 右击添加类，把当前类与ui关联

  6. 在主对话框中

    1. 第一种方法：双击按钮，自动创建点击鼠标事件。
    2. 第二种方法：右击button属性，选中控件组件，有事件，添加
    3. 第三种方法：右击button，插入ActiveX控件

  7. 在其中类中，进行引入需要弹出的对话框关联的类，进行操作

非模态对话框

  • 1~6与上面一样。

  • 在其中类中，进行引入需要弹出的对话框关联的类，进行操作

  1. 新建对话框项目

  2. 添加工具箱中的Static text

  5. 可以做事件操作了

    2. ctrl +左键拖拽 可以复制编辑框

    3. 退出对话框

    4. 新建编辑框，点回车会退出整个对话框

      1. 类视图中DLG右击—属性–点击重写–onOK–添加

      2. 注释掉

  4. 添加成员变量为value(一般只有编辑框才设为value)

    1. 设置编辑框的内容

    2. 读取编辑框的内容

SpringBoot整合RabbitMQ

SpringBoot整合RabbitMQ

Exchange类型

  • 生产者把消息发布到Exchange上，消息最终到达队列并被消费者接受，binding决定交换器的消息应该发送到哪个队列

direct

  • 消息中的 路由键如果和binding中binding key一致，交换器就把消息发送到对应的队列中。
  • 它是完全匹配，单播的形式

Fanout

  • 把下面的所有队列全部发送到对应的队列中
  • 转发消息是最快的

Topic

  • 把路由键和绑定键的字符串切分成单词，这些单词之间有点隔开
  • 队列的binging key，通常是#.name,name.#
  • #表示0个或多个单词，*匹配一个单词

Docker中下载RabbitMQ

  • docker pull rabbitmq:3-management

  • docker images

  • docker run -d -p 5672:5672 -p 15672:15672 –name myrabbitmq 83878773002b

  • 网页访问15672端口就可以进入后台，默认账号和密码都是guest
